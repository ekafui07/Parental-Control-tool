=====================================================
  PARENTAL CONTROL - HOW TO INSTALL
=====================================================

STEP 1: Install .NET 8 SDK (if not already installed)
  → https://dotnet.microsoft.com/download/dotnet/8.0
  → Download the SDK (not just Runtime)
  → Install it and restart your computer

STEP 2: Double-click INSTALL.bat
  → Click "Yes" when Windows asks for Admin permission
  → A PowerShell window will open
  → Wait ~1-2 minutes for it to build and install
  → You'll see green [OK] messages if successful

STEP 3: Check for errors
  → If the installer stops or shows red [FAILED] messages:
     - Open "install.log" in this folder
     - Read the error messages
     - Common issues:
       * .NET 8 SDK not installed (go back to Step 1)
       * Antivirus blocking the installer (disable temporarily)
       * Another parental control app running (uninstall it first)

STEP 4: Done!
  → A "Parental Control" shortcut appears on your Desktop
  → Blocking starts IMMEDIATELY — no setup needed
  → Service runs 24/7 in background
  → Desktop shortcut opens the management GUI for monitoring and admin tasks

HOW IT WORKS (MULTI-LAYERED PROTECTION)
=====================================================

Layer 1: Hosts File (Domain Blocking)
  • Rewrites system hosts file every 30 seconds
  • Blocks all hardcoded domains (porn, torrent sites, etc.)
  • Keyword filtering (porn, xxx, sex, cam, etc.)
  • Cannot be bypassed by:
    - Clearing browser history/cookies
    - Using Incognito/Private mode
    - Editing hosts file manually (restored every 30 sec)

Layer 2: HTTP Proxy (Video File Blocking)
  • Intercepts HTTP traffic on port 8080
  • Detects and blocks all video file downloads
  • Checks by: file extension, MIME type, and magic bytes
  • Blocks 40+ video formats (.mp4, .mkv, .avi, .mov, etc.)
  • Returns 403 Forbidden for blocked video files

Layer 3: Browser Extension Blocking
  • Disables all existing extensions
  • Prevents installation of new extensions via Group Policy
  • Monitored and re-enforced every 60 seconds
  • Works in: Chrome, Edge, Firefox, Opera, Brave, Vivaldi

Layer 4: Process Killer
  • Terminates VPN clients (ProtonVPN, NordVPN, etc.)
  • Kills Tor Browser and Brave on startup
  • Blocks proxy tools (Psiphon, UltraSurf, Lantern, etc.)
  • Checks every 10 seconds for bypass attempts

Whitelisting Feature:
  • Some legitimate sites may be blocked by keyword matching
  • Use the GUI to whitelist sites like Reddit with admin password
  • Whitelisted domains bypass all keyword filtering

Cannot be stopped by:
  • Regular users (requires admin access)
  • Clearing cache or browsing in incognito mode
  • Editing hosts file (auto-restores)
  • Changing DNS (hosts file still blocks)
  • Installing bypass tools (processes killed automatically)

Can ONLY be stopped by:
  • Uninstalling service (requires admin password: n0Zone2017)
  • Stopping Windows Service in Services.msc (requires admin)

=====================================================
TROUBLESHOOTING
=====================================================

Problem: Installer stops immediately after admin prompt
  → Check install.log for errors
  → Most likely: .NET 8 SDK not installed

Problem: Sites still accessible after install
  → Open Desktop shortcut
  → Check if service status shows "Running"
  → If not running, try rebooting computer

Problem: Can't add/remove sites in manager
  → You need the admin password
  → Password: n0Zone2017 (hardcoded, cannot be changed)
  → Same password for all admin functions (add, remove, whitelist)

Problem: A legitimate site is blocked (e.g., Reddit)
  → Use the GUI to whitelist it
  → Open "Parental Control" from Desktop
  → Go to "Active Blocks" tab → click "Whitelist Domain"
  → Enter the domain and admin password
  → The domain will be allowed even if it matches keywords
  → Whitelist is saved and persistent

Problem: Want to temporarily disable blocking
  → Open Services (services.msc)
  → Find "ParentalControlService"
  → Stop it (requires admin access)
  → Note: This is not recommended for production use
  → Start it again when needed (double-click → Start)
  → Service will auto-start on next reboot unless disabled
